# baseline_v1.1 변경사항 (v1 → v1.1)

기준: v1(yolo11m / 1280 / 60ep / cos_lr / patience60) → Kaggle **0.97**

## 목적
**bbox 탐지·분류 성능을 모델 용량으로 끝까지 짜낸다.** 단일 모델 최대치를 본 뒤, 다음 단계(클래스 커버리지 보강)로 넘어가기 위한 기준선.

## 무엇을 바꿨나

| 항목 | v1 | v1.1 | 이유 |
| --- | --- | --- | --- |
| `--model` | yolo11m | **yolo11x** | 최대 용량 모델. 분류·박스 정밀도 향상 여지 |
| `--epochs` | 60 | **150** | 모델이 커지고 데이터는 작아(518장) 충분히 수렴시키려고 |
| `patience` (train_yolo.py) | 60 | **200** | epochs(150)보다 크게 → **조기종료 사실상 없음**. 끝까지 학습. best.pt는 최고 시점 자동 저장 |
| `--batch` | 8 | 8 | A100(40GB) 기준. OOM 시 4로 |
| `cos_lr` | True | True | 유지 |
| 증강 | 전부 OFF | 전부 OFF | train≈test 동일분포라 유지 |
| 추론 | conf 0.15 / iou 0.6 / max_det 4 | 동일 | 유지 |

## GPU
- **A100 (40GB)** 전제. yolo11x @ 1280 batch8은 A100에서 동작 (T4 16GB로는 불가, OOM).
- OOM 시: `--batch 4`. 그래도 안 되면 `--imgsz 1024`.

## 주의 — 이 실험의 한계 (중요)
test에는 **train에 없는 클래스**가 존재 (test 40종 ⊄ train 56종). 미학습 클래스는 **모델을 아무리 키워도 분류 불가** → yolo11x로도 그 부분 점수는 못 올림.
- 즉 v1.1은 "탐지/박스 + 아는 클래스 분류"의 상한을 보는 실험.
- 미학습 클래스 문제는 **다음 작업(AIhub에서 빠진 클래스만 받아 train 보강)**에서 해결. test 40종 category_id 목록 확보가 선결 과제.
