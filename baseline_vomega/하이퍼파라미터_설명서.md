# baseline_vomega 하이퍼파라미터 설정 위치

3-모델 앙상블: **yolo11x + yolo26x (학습 완료) + RT-DETR (이 노트북에서 학습)**

거의 모든 설정은 `colab_run.ipynb` **셀 2 (여기만 수정)** 한 곳에 모여 있습니다.

## 1. 셀 2에서 바로 바꾸는 것

| 변수 | 기본 | 설명 |
| --- | --- | --- |
| `RTDETR_MODEL` | `rtdetr-l.pt` | RT-DETR 크기. 더 키우면 `rtdetr-x.pt` |
| `RTDETR_EPOCHS` | 150 | RT-DETR 에폭 (DETR 계열은 수렴 느림 → 넉넉히) |
| `RTDETR_IMGSZ` | 1024 | RT-DETR 입력 해상도 (1280은 ~2배 느림) |
| `RTDETR_BATCH` | 16 | A100 80GB 기준. 여유로 24, 또는 -1(자동) |
| `WBF_WEIGHTS` | `'2 2 1'` | 앙상블 가중치 [yolo11 yolo26 rtdetr] |
| `WBF_IOU` | 0.55 | WBF 박스 병합 IoU |
| `RTDETR_SUBMISSION` / `FINAL_SUBMISSION` | csv 이름 | 출력 제출본 파일명 |
| `YOLO11_NAME` / `YOLO26_NAME` | `yolo11.pt` / `yolo26.pt` | Drive `outputs/yolo/weights/`의 가중치 파일명 |
| `RESUME` | False | True면 Drive 백업 last.pt에서 RT-DETR 이어 학습 |

## 2. RT-DETR의 lr / 스케줄러 / optimizer

`train_rtdetr.py`의 `model.train(...)` 호출에 인자를 추가하면 됩니다. 예:

```python
model.train(
    ...
    lr0=0.0001,      # RT-DETR은 YOLO보다 작은 lr 권장
    lrf=0.01,
    cos_lr=True,     # 이미 켜져 있음
    optimizer='AdamW',
    weight_decay=0.0001,
    warmup_epochs=3,
)
```
전체 옵션: https://docs.ultralytics.com/modes/train/#train-settings

## 3. 추론 / 앙상블 인자

- **RT-DETR 단독** (셀 11, `inference_rtdetr.py`): `--conf 0.15 --iou 0.6 --max_det 4`
- **앙상블** (셀 12, `inference_ensemble_vomega.py`): `--weights`, `--wbf_iou`, `--yolo_imgsz 1280`, `--rtdetr_imgsz`, `--conf`, `--iou`, `--max_det`
  - YOLO 2개는 TTA(augment) 켜짐, RT-DETR은 미지원이라 꺼짐
